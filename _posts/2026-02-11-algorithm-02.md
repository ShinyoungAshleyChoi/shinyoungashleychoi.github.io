- 이론 정리
    - 브루트 포스 알고리즘
        - 모든 경우를 살펴 봄으로써 답을 구하는 알고리즘
        - 수행 시간이 오래 걸리므로 시간 제한을 고려하여 선택하여야 함
            - 소수 판별 문제
                - 브루트 포스 사용시 O(N^2) or O(NlogN)
                - 에라토스테네스의 체 사용시 O(N log(logN))
    - 그리디 알고리즘
        - 매 순간에서 가장 최선인 선택을 하여 답을 구함
        - 매 순간에서의 최선의 선택 == 문제에서의 최적해 인 경우에 사용 가능
        - 정렬 후 위 조건을 만족하는 경우가 있으므로 정렬과 함께 사용되는 경우가 많음
    - DP 알고리즘 - 실제로 “다이나믹”과는 연관이 없음
        - 복잡한 문제를 간단한 여러 개의 문제로 나누어 푸는 기법
        - 메모이제이션을 활용하여 중복된 계산을 줄임 → DP table에 저장
        - DP 알고리즘 구현 방법
            1. 문제의 재귀적인 구조 파악
            2. 수식으로 표현
            3. 초기값 처리
        - Top down 방식과 Bottom up 방식
            - Top down: 큰 부분(문제에서 구하고자 하는 값)을 기준으로 필요한 부분만 구하는 방식
            - Bottom up
                - 작은 부분부터 완성하는 방식, 보통 반복문을 이용
                - 전체를 구해야할 때 유용
                - DP 테이블을 갱신하는 과정에서 순서에 유의. 아직 갱신하지 않은 값을 참조할 수 없음

- BOJ2503 (https://www.acmicpc.net/submit/2503)
    - **유형:** #브루트포스 #조합
    - **난이도(선택):**
    - **시간복잡도**
        - O(9P3 * N) → O(N)
        - 시간복잡도 근거: 1~9 중에 3가지를 뽑는 조합의 개수만큼 반복하면서 정답인지 아닌지 확인해보면 됨
    - **아이디어**
        - **핵심 풀이 아이디어:** 시간복잡도가 O(N)이고 제약 조건이 111 ≤ N ≤ 999 이므로 모든 경우의 수를 확인하는 브루트 포스를 사용해볼만함
        - **사고 과정:**
            1. 1~9중에 3가지를 뽑는 조합을 다 뽑는다
            2. 문제 조건에 맞는지 확인해서 정답일 가능성을 체크한다.
        - **엣지 케이스(선택):**
    - **구현 순서**
        1. 입력 받기
        2. 조합 나열하여 for문 돌리기
        3. 문제 조건에 맞는지 확인하기
        4. 문제 조건에 맞지 않으면 넘어가고 맞으면 정답 개수를 추가시키기
    - **구현 코드**

        ```python
        from itertools import permutations
        
        N = int(input())
        condition = [input().split() for _ in range(N)]
        
        answer = 0
        for p in permutations(range(1, 10), 3):
        	ok = True
        
        	for num, st, b in condition:
        
        		p_st = p_b = 0
        		for i in range(3):
        			if int(num[i]) == p[i]:
        				p_st += 1
        			elif int(num[i]) in p:
        				p_b += 1
        
        		if p_st != int(st) or p_b != int(b):
        			ok = False
        			break
        			
        
        	if ok:
        		answer += 1
        
        print(answer)
        ```

    - **회고:** 처음에 ok를 False로 두고 조건 (p_st == int(st) and p_b == int(b))이 맞으면 ok를 True로 변경하였는데 이렇게 하면 문제 조건 중에 하나만 만족해도 정답 후보로 카운팅 됨. ok를 True로 초기화하고 한번이라도 조건에 맞지 않을 때 False로 변경하는 것이 간단하고 좋음
- BOJ5585
    - **유형:** #그리디
    - **난이도(선택):**
    - **시간복잡도**
        - O(1)
        - 시간복잡도 근거: 풀이참고
    - **아이디어**
        - **핵심 풀이 아이디어:** 500 100 50 10 5 1 순으로 우선순위가 명확하므로 그리디한 풀이가 가능
        - **사고 과정:**
            1. 받을 거스름 돈을 구하고
            2. 동전의 최 우선순위부터 거스름돈에서 나눈 몫을 과 나머지를 구함
            3. 위에서 나온 나머지를 가지고 다음 동전으로 나눔
            4. 반복 …
        - **엣지 케이스(선택):**
    - **구현 순서**
        1. 입력 값 받기
        2. 받아야 할 돈은 1000 - N
        3. 동전을 우선순위대로 나열
        4. 연산
    - **구현 코드**

        ```python
        price = int(input())
        change = 1000 - price
        
        coins = [500, 100, 50, 10, 5, 1]
        
        cnt = 0 
        for coin in coins:
        	cnt += change // coin
        	change %= coin
        
        print(cnt)
        ```

- BOJ1149 (https://www.acmicpc.net/problem/1149)
    - **유형:** #DP
    - **난이도(선택):**
    - **시간복잡도**
        - O(3N)
        - 시간복잡도 근거: 풀이참고
    - **아이디어**
        - **핵심 풀이 아이디어: DP를 이용**
        - **사고 과정:**
            1. 브루트포스를 사용하기에는 3 X 2^n-1 번 돌아야하기 때문에 어려움
            2. 그리디한 풀이를 하기에는 현재 최적의 값이 문제의 최적해가 아님
            3. DP로 풀 수 있음
        - **엣지 케이스(선택):**
    - **구현 순서**
        1. 입력 받기
        2. DP 테이블 초기화
        3. 인덱스 2부터 전 값의 최소값을 받아와 DP 테이블 갱신
    - **구현 코드**

        ```python
        N = int(input())
        
        cost = [[0, 0, 0]]
        cost += [list(map(int, input().split())) for _ in range(N)]
        
        dp = [[-1, -1, -1] for _ in range(N+1)] 
        dp[1][0] = cost[1][0]
        dp[1][1] = cost[1][1]
        dp[1][2] = cost[1][2]
        
        for i in range(2, N+1):
        	dp[i][0] = cost[i][0] + min(dp[i-1][1], dp[i-1][2])
        	dp[i][1] = cost[i][1] + min(dp[i-1][0], dp[i-1][2])
        	dp[i][2] = cost[i][2] + min(dp[i-1][0], dp[i-1][1])
        
        print(min(dp[N]))
        
        ```

    - **회고:** 어려움.. 이게 레벨 1이라니..
- BOJ1182 (https://www.acmicpc.net/problem/1182)
    - **유형:** #브루트포스 #조합
    - **난이도(선택):2**
    - **시간복잡도**
        - O(2^n)
        - 시간복잡도 근거: 부분 집합의 개수가 2^n임
    - **아이디어**
        - **핵심 풀이 아이디어: 브루트포스를 이용**
        - **사고 과정:**
            1. 부분집합의 개수는 2^n이고 1 ≤ N ≤ 20, |S| ≤ 1,000,000 이므로 최대 2^20 == 1000000 ≤ 1억 → 브루트포스를 이용해 풀 수 있다.
        - **엣지 케이스(선택):**
    - **구현 순서**
        1. 입력받기
        2. 모든 부분집합을 나열해 합을 구하기
        3. S와 일치하면 break
    - **구현 코드**

        ```java
        
        from itertools import combinations
        
        N, S = list(map(int, input().split()))
        set_ = list(map(int, input().split()))
        
        answer = 0
        for n in range(1, N+1):
        	for c in combinations(set_, n):
        		if S == sum(c):
        			answer += 1
        
        print(answer)
        ```

    - **회고: 처음에 크기가 0인 부분 수열까지 생각을 못함**
- BOJ1026 (https://www.acmicpc.net/problem/1026)
    - **유형:** #그리디
    - **난이도(선택): 4**
    - **시간복잡도**
      - 
        - 시간복잡도 근거:
    - **아이디어**
        - **핵심 풀이 아이디어:**
        - **사고 과정:**
            1. 브루트포스로 풀 수 있을까?
            2. NPN → 최악의 경우 50P50 == 50! = 엄청큼.. 안됨.
            3. 큰수에 가능한 작은 수를 곱하면 됨
        - **엣지 케이스(선택):**
    - **구현 순서**
        1. 입력 받기
        2. 재정렬하기 → 문제에서는 B를 재정렬하지 말라고 하지만 A를 출력하는 것이 아니라 S를 출력하는 것이므로 재정렬해도 상관 없음
        3. 반복문으로 요소들 곱해서 다
    - **구현 코드**

        ```python
        
        N = int(input())
        A = list(map(int, input().split()))
        B = list(map(int, input().split()))
        
        A = sorted(A)
        B = sorted(B, reverse=True)
        
        ans = 0
        for a, b in zip(A, B):
        	ans += a*b
        
        print(ans)
        ```

    - **회고: B를 재정렬하지 말래서 처음에 어렵게 접근함..**
