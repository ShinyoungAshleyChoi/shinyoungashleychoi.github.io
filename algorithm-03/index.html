알게된 내용 정리

  • 이론 정리
    • 이분 탐색 알고리즘
      • 정렬된 배열에서 특정한 원소를 찾는 알고리즘
      • 정렬이 가능한 상황이며 선형자료구조에서 사용을 고려
      • O(N) (선형탐색) → O(logN) (이분탐색)
        • 방법1: 일반적인 이분탐색
        • 방법2: 파라매트릭 서치를 이용한 방법
    • 파라매트릭 서치 알고리즘
      • 최대값, 최소값을 구하는 문제를 결정 문제(OX로 답할 수 있는 질문형태의 문제)로 바꾸어 푸는 알고리즘
    • 투포인터 알고리즘
      • 두 개의 포인터를 이용하여 문제를 푸는 알고리즘
      • 1 차원 배열이 나오고 답의 후보를 효율적으로 탐색해야할 때 많이 쓰임

  • BOJ1920 (https://www.acmicpc.net/problem/1920)
    • 유형: #이분탐색
    • 난이도(선택):
    • 시간복잡도
      • O(N+N logN)
      • 시간복잡도 근거: 아래 참고
    • 아이디어
      • 핵심 풀이 아이디어: 이분 탐색
      • 사고 과정:
        1. 브루트포스로 풀 수 있을까?
          1. 선형 탐색하는 경우
            1. O(M) * O(N) → O(MN): 10만 * 10만 → 100억
          2. 정렬 후 이분 탐색
            1. 정렬: O(NlogN) + 이분 탐색: O(MlogN) = O(N+M logN)
          3. set을 활용한 풀이: O(N + M)
      • 엣지 케이스(선택):
    • 구현 순서
      1. 입력 받기
      2. M 값을 순회하면서 N에 존재하는지 확인하기
      3. 이분 탐색으로 is_exist 구현하기

    • 구현 코드

        N = int(input())
  arr = sorted(list(map(int, input().split())))
        
  M = int(input())
  nums = list(map(int, input().split()))
        
  def is_exist(arr, num):
        
  	left = 0
  	right = len(arr) - 1
        
  	while left <= right:
  		mid = (left + right) // 2
        
  		if arr[mid] < num:
  			left = mid + 1
        
  		if arr[mid] > num:
  			right = mid - 1
        
  		if arr[mid] == num:
  			return 1
        
  	return 0
        
  for num in nums:
  	print(is_exist(arr, num))

  • BOJ2110 (https://www.acmicpc.net/problem/2110)
    • 유형: #파라매트릭서치
    • 난이도(선택):
    • 시간복잡도
      • O(30 N)
      • 시간복잡도 근거: 답의 후보에 대하여 파라매트릭서치로 풀 수 있음
    • 아이디어
      • 핵심 풀이 아이디어: 파라매트릭서치
      • 사고 과정:
        1. 최소값의 최대값을 구하는 문제이므로 파라매트릭서치로 접근
      • 엣지 케이스(선택):
    • 구현 순서
      1. 첫 집에 공유기 1개 설치
      2. 그 다음부터는 마지막으로 설치한 집 좌표 기준으로 x[i] - last ≥ d 이면 설치
      3. 설치 개수가 C 이상이면 d는 가능

    • 구현 코드

        import sys
  input = sys.stdin.readline
        
  N, C = map(int, input().split())
  x = [int(input()) for _ in range(N)]
  x.sort()
        
  def can_place(d: int) -> bool:
      cnt = 1
      last = x[0]
      for i in range(1, N):
          if x[i] - last >= d:
              cnt += 1
              last = x[i]
              if cnt >= C:
                  return True
      return False
        
  low, high = 1, x[-1] - x[0]
  ans = 0
        
  while low <= high:
      mid = (low + high) // 2
      if can_place(mid):
          ans = mid
          low = mid + 1
      else:
          high = mid - 1
        
  print(ans)

  • BOJ1806 (https://www.acmicpc.net/problem/1806)
    • 유형: #투포인터
    • 난이도(선택):
    • 시간복잡도
      • O(N)
      • 시간복잡도 근거:
    • 아이디어
      • 핵심 풀이 아이디어: 투포인터
      • 사고 과정:
        1. 브루트포스로 풀 수 있을까?
          1. 누적합으로 풀어도 N^2 이므로 시간제한내에 풀 수 없음
        2. 모든 시작점에 대해 적절한 끝점을 투포인터로 계산
      • 엣지 케이스(선택):
    • 구현 순서
      1. l=0, sum=0, ans=무한대
      2. r을 0..N-1까지 늘리면서 sum += a[r]
      3. sum >= S 인 동안:
        • ans = min(ans, r-l+1)
        • sum -= a[l], l += 1 (구간을 줄여서 더 짧게 가능한지 확인)
      4. 끝났는데 ans가 갱신 안 됐으면 0 출력, 아니면 ans

    • 구현 코드

        import sys
  input = sys.stdin.readline
        
  N, S = map(int, input().split())
  a = list(map(int, input().split()))
        
  l = 0
  cur = 0
  INF = 10**9
  ans = INF
        
  for r in range(N):
      cur += a[r]
        
      while cur >= S:
          ans = min(ans, r - l + 1)
          cur -= a[l]
          l += 1
        
  print(0 if ans == INF else ans)
